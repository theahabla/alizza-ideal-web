### ■　欲しかったもの　■　それは、こんな Web サイト

ここで説明するWebサイトの作り方は、わたしと同じ次のような人のニーズを満たします。

> - カスタムメイドのデザイン
- ブログや作品、商品の説明文と画像を簡単に追加できる
    - もちろん変更も簡単にできる
- クレジットカード決済が使え、商品の販売ができる
- 毎月の維持費用 500円以下

すでに信じられませんか。特に、最後のランニングコストが500円以下だなんて。

でも、今あなたが見ている [このサイト](https://www.alizza-ideal.com/) が
それを証明しています。

最初の2つは、当たり前のニーズですよね。

WordPress であれば、無料のプラグインを追加することで、
ある程度は簡単に実現できます。
それに加え、HTML や PHP などのプログラミング言語を、
ちょっとの労力と時間を使って学べば、もっと自由にデザインできます。

それと同じような、カスタムメイド・デザインの Web サイトがほしかったのです。

ではなぜ、WordPressではないのか。それは、後半で説明します。

### ■　なぜ独自サイト？　■　楽天ではだめなの？

ネットで販売するには、独自サイトを使わずに、
すでにある Amazon とか楽天市場を利用するのが簡単です。

今やメルカリも、個人以外が商売として参加することができるようになりました。

ただ問題は、そのようなネットモール、ポータルサイトでは、
目立つのが難しい点にあります。

え、逆じゃないの？と、思うかもしれません。

確かに、ポータルサイトに登録すれば、
ポータルサイトの知名度によって、
そこには多くのお客さん候補（潜在客）が集まります。

しかし、ポータルサイトやネットモールには、
あなたのお店以外にも、多くのお店が出店しています。

その数は、現実のショッピングモールに入っている
お店の数の比ではありません。

では、SEOの出番だと思いますよね。

しかし、世の中に出回っているSEOの方法は、
Googleの検索エンジン向けです。

Yahoo!ショッピングの SEO の専門家は、聞いたことありますか？
もし、いるとしてもそんなに多くありません。

では、仮にお客さんがあなたの商品ページを見たとします。

しかし、たとえば Amazon とかは、他のお店の商品を勧めてしまいます。
もっと安い商品がありますとか、他の人は別のを買っていますとか。

みんなが買っているものがあると言われたら、そっちのほうが気になりますよね。

これって、いうなればお店を借りているビルのオーナーさんが、
あなたのお店の前で他のお店の客引きをしているようなものです。

平気でいられますか？

そこで、今ネットで商売をするなら、独自サイトが欠かせないというわけです。


### ■　初期費用が安いだけじゃヤダ　■　毎月の維持費用 500円以下
小さな会社やお店にとって、毎月の固定費の負担は死活問題です。
とくに、このコロナ禍にあっては。

毎月の維持費用は少ないにこしたことはありません。
それでなくても、家賃や光熱費、人を雇っているなら人件費と、
固定費は重くのしかかってきます。

さて実際には。

今の所、独自ドメインの維持費用だけで済んでいます。
Google Domain で１年契約で取得したので、
月200円もかかっていません。

Webサイトをホスティングするサーバーは、
無料プランを使っているので、ゼロ円です。

もちろん、アクセスが集中したり、
ディスクがいっぱいになったりすれば、
無料プランでは間に合わなくなります。

ただこの先、そのくらい商売がうまくいったなら、
余裕を持って有料プランに移行できるはずです。

今は、はやく有料プランに切り替えようという、
モチベーションアップになっています。

それもあって、サイトで何か商品を販売することにしました。

### ■　わたしが WordPress じゃ嫌だったわけ　■

わたしは、子供の頃からあまり裕福ではなかったので、
今でもお金をケチることが染み付いています。

さて、以前より聞いていました。
独自サイトを作るなら、WordPressを使えば無料で作れると。

しかし、猜疑心の強いわたしは、飛びつく前に、本当にそうか調べてみました。

すると、WordPress自体は無料で使えるのはわかりました。
しかし、WordPressを使うためには、それに付随する費用が
かかることもまた、わかりました。

その費用は、金銭的な費用だけでなく、
時間的な費用、つまりわたしの労働時間も含まれます。

わたしの知り合いからも、WordPress にとても苦労していると聞きました。
もっとも、その人はプログラマではないので、さもありなんではあるのですが。

ただ、ハードワークの合間に、片手間で構築から更新までするなんて、
できそうもないと思いました。

> - レンタルサーバー代
- セキュリティー対策代
- バックアップ代
- カスタマイズ用のプラグイン費用
- プラグインのバージョンアップ保守費用
- WordPressを使いこなすための勉強費用

一度設定してしまえば簡単、という人もいます。

しかし、元来凝り性のわたしは、深いところまで手を加えたり、
中を覗かなくてはいられないたちなのです。

ある意味、プログラマの悪癖ですね。

設定したままにすることなんて、想像できませんでした。

以前、お名前.com で借りたサーバーに、
試しに WordPress を導入して見たこともあります。

そしたら、お名前.com から、WordPress プラグインの更新メッセージが、
毎日毎日、大量に届き出しました。

WordPress の保守が大変というイメージは、このときに植え付けられました。

また、別の知人からも時折、 WordPress がクラッシュしてデータがなくなったとか、
ウイルスにやられたとか、聞かされています。

そして、今年のはじめに試してみようと、家の NAS に WordPress を入れてみました。
全然試している暇がなく、ずっと忘れていました。

ところが、先月気がついたのです。
NAS にバックアップしていた写真ファイルとテキストファイルが、
すべてランサムウェアによってパスワード付きで圧縮されていました。

もちろん、侵入経路は判明しておらず、WordPress との関連は不明です。

あなたの WordPress は大丈夫ですか？

### ■　最終的に JAMstack を選択した　■

そのため、知識をもとめてネットをさまよう日がしばらく続きました。

そしてとうとう、自分にあった、納得のいく解決策が見つかったのです。

それが、JAMstack と呼ばれる、 Web サイトを構築するための方法論でした。

さて、少し長くなりましたので、続きは後日に投稿します。

具体的に、どのように構築したのかを説明するつもりです。
